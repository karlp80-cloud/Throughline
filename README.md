# Throughline

A Zachtronic-style flow-routing puzzle game whose mechanics are fixed but whose themes, narrative, and puzzle layouts are generated by an LLM. Two artifacts:

- **Game** — TypeScript / HTML5 Canvas. Runs in the browser (Vite) and as a desktop app (Tauri 2.x).
- **`throughline-gen`** — Node CLI that spawns `claude -p`, validates the result against a strict Zod schema, runs an automated solvability check, and writes a `campaign.json` manifest the game consumes.

The CLI is the only component that talks to an LLM. The game is a pure function of `campaign.json`.

See [throughline-design.md](throughline-design.md) for the full design and [IMPLEMENTATION_PLAN.md](IMPLEMENTATION_PLAN.md) for the phased build plan.

## Installation

### Desktop (recommended)

Download the installer for your platform from the [Releases page](https://github.com/karlp80-cloud/Throughline/releases) once `v0.1.0` is tagged.

The desktop app's procedural-generation flow (the "New Campaign" button) requires two things on your PATH:

1. **Node ≥20** — [nodejs.org](https://nodejs.org/).
2. **Claude Code CLI (`claude`)** — install per [Anthropic's Claude Code docs](https://docs.claude.com/en/docs/claude-code/overview). The game spawns `claude -p` to generate campaigns; without it, only the bundled sample campaigns are playable.

If either is missing, the New Campaign modal explains what to install.

### Browser

A hosted browser build will be linked from the Releases page once `v0.1.0` ships. The browser build has no procedural-generation button — it offers a file picker for `campaign.json` manifests you generated with the CLI elsewhere.

## Sample campaigns

The desktop bundle ships with hand-crafted sample campaigns under `campaigns/samples/` so the game is playable without `claude` installed. The main menu lists them under "Generated campaigns".

In the browser build, load a sample by:

1. Generating one with the CLI: `npx throughline-gen --out my-campaign.json --seed lighthouse --acts 1 --puzzles-per-act 3`, or
2. Cloning the repo and pointing the browser file picker at any file under `campaigns/samples/`.

## Running from source

Prerequisites:

- **Node ≥20** (`npm` ships with it).
- For the desktop build only: **Rust toolchain** via [rustup](https://rustup.rs/) plus the [Tauri 2.x system prerequisites](https://v2.tauri.app/start/prerequisites/) for your OS. On Windows that's the Microsoft C++ Build Tools and WebView2 (usually preinstalled on Win10+).

```
git clone https://github.com/karlp80-cloud/Throughline.git
cd Throughline
npm install
```

### Browser dev / build

```
npm run dev               # local dev server at http://localhost:5173
npm run build:web         # production browser bundle into dist/
npm run preview           # serve dist/ at http://localhost:4173
```

The browser path has zero Rust dependency.

### Desktop dev / build

```
npm run tauri dev         # launches the desktop app with hot reload
npm run build:desktop     # production bundles (.msi/.nsis on Win,
                          #   .dmg on macOS, .deb/.AppImage on Linux)
npm run build:desktop:debug   # same, but unstripped — faster to smoke
```

The first `tauri dev` or `tauri build` on a clean machine takes several minutes to compile the Tauri Rust dependencies; subsequent builds are incremental.

### CLI

```
npm run build:cli                                         # compile to dist-cli/
node bin/throughline-gen --out out.json --seed demo \\
  --acts 1 --puzzles-per-act 3                            # generate a campaign
```

Run `node bin/throughline-gen --help` for full flag documentation. The CLI requires `claude` on PATH.

### Tests

```
npm run lint              # ESLint + Prettier check
npm run test:unit         # Vitest (Node + jsdom)
npm run test:e2e          # Playwright (auto-builds via pretest:e2e)
npm test                  # unit + e2e
cargo test --manifest-path src-tauri/Cargo.toml    # Rust-side tests
```
